# Office Software 操作界面功能说明

更新日期：2026-07-02

本文档说明 `office-software` 本地操作平台的启动方式、现有功能、字段含义、执行流程和注意事项。之后如果继续新增、删除或调整操作界面功能，需要同步更新本文档。

## 启动方式

在项目目录 `E:\office-software` 中启动：

```powershell
npm start
```

启动成功后访问：

```text
http://localhost:3217/inline-packager.html
```

当前服务端入口是 `server.js`，前端页面在：

```text
E:\office-software\办公软件\111\inline-packager.html
```

## 公共说明

- 页面是本地工具平台，主要面向网站内容、资料上传、Banner / Popup 后台配置和巡查。
- 部分后台自动化会打开真实浏览器执行操作，执行期间不要关闭自动打开的浏览器窗口。
- 商城后台账号默认从桌面的 `网站账号密码（HQ数字营销组使用）.xlsx` 中按站点读取；页面里的账号密码输入框可作为手动覆盖。
- 涉及真实后台提交的按钮通常带有“执行”字样；“生成清单”类按钮只生成结果，不会提交后台。
- 输出区域会显示执行日志、生成结果、巡查结果或错误信息。

## 一键内联打包

用于把本地页面素材处理成可上线的内联 HTML。

主要字段：

- `CSS 兼容模式`：选择输出兼容版本。
- `图片处理模式`：决定图片是否上传、替换或保持原样。
- `图片基础地址`：默认 `https://mfs.ezvizlife.com/`。
- `上传接口`：默认 `https://fs.ezvizlife.com/upload.php`。
- `选择项目文件夹`：选择包含 HTML 和素材的文件夹。
- `选择 HTML 文件`：从识别到的 HTML 文件中选择要处理的页面。

主要按钮：

- `开始一键内联`：执行处理并在输出框显示处理后的 HTML。

## PDF 上传转地址

用于把 PDF 上传到文件服务，并得到可复制的线上地址。

主要字段：

- `上传接口`：默认 `https://fs.ezvizlife.com/upload.php`。
- `选择 PDF`：支持选择一个或多个 PDF。

主要按钮：

- `开始上传`：上传 PDF。
- `复制地址`：复制上传后的地址列表。

## Spec 参数解析

用于把 Excel 中的规格参数转换为前端可用的 PC / Mobile HTML。

主要字段：

- `Excel 文件`：选择 `.xlsx` 或 `.xls`。
- `固定参数数量`：控制固定展示的参数数量。
- `语言`：选择输出语言。
- `标题`：默认 `Specifications`。
- `图片地址`：规格区域使用的图片 URL。
- `图片 Alt`：图片替代文本。

主要按钮：

- `生成`：生成 PC 和 Mobile HTML。
- `复制 PC`：复制 PC 输出。
- `复制 Mobile`：复制 Mobile 输出。

## i18n 语言转换

用于把文案转换为 i18n 结构，也支持解析已有语言表。

主要字段：

- `语言 Key 前缀`：例如 `goods.Alarm_light`。
- 表格区域：可在“转换表”和“已有表”之间切换。

主要按钮：

- `解析并提取`：从输入内容中提取语言 key 和文案。
- `导出 i18n 文件`：导出转换后的语言文件。
- `解析表格`：解析表格内容。

## 下载链接查询

用于查询产品相关下载资料，并生成前端下载链接。

功能区域：

- `下载生成器`：根据产品名称查询并生成下载链接。
- `手动生成下载链接`：手动输入信息生成并打开下载链接。
- `API Headers`：用于存放请求所需 header。

主要按钮：

- `查询并生成`
- `生成并打开`

## 资料上传平台

用于把产品资料上传到服务中心后台，并生成相关归档计划。

典型流程：

1. 填写产品标题，例如 `EP8 Ultra`。
2. 关联产品默认按产品标题查找，也可填写产品搜索词。
3. 选择文件类型，例如 `Product Datasheet`。
4. 选择状态、权重。
5. 选择资料文件夹或多份文件。
6. 系统自动识别：
   - 文件名包含“高清图”的图片作为高清图。
   - 文件名包含 datasheet 或 PDF 类型的文件作为 datasheet。
   - Spec Excel 按文件类型识别。
7. 按需勾选创建下载资料、扩展语言、更新产品背景图和 SharePoint 归档计划。
8. 点击 `开始执行`。

主要字段：

- `产品标题`：后台标题，通常使用产品名。
- `关联产品搜索词`：默认使用产品标题。
- `文件类型`：下载中心资料类型。
- `状态`：是否启用。
- `权重`：排序权重。
- `后台账号 / 后台密码`：已登录可留空。
- `选择产品资料文件夹`：可一次选择产品资料目录。
- `选择零散文件`：可选择多个独立文件。

勾选项：

- `生成 SharePoint 归档计划（不上传）`
- `创建下载中心资料`
- `访问扩展接口生成语言`
- `更新产品背景图`

SharePoint 归档字段：

- `翻译资料根目录`：默认 `Shared Documents/05_Website/00_Product Translation`。
- `素材根目录`：默认 `Shared Documents/05_Website`。
- `素材类目`：选择或自定义素材分类文件夹。

主要按钮：

- `开始执行`：执行所选动作。
- `仅识别文件`：只识别文件类型，不提交后台。
- `清空日志`：清空日志区域。

注意事项：

- 当前 SharePoint 相关能力以归档计划为主，不等同于一定直接上传。
- 文件名中建议明确包含 `高清图`、`datasheet`、`spec` 等关键词，便于自动识别。

## Banner / Popup / 巡查

这是活动后台配置和首页活动链接巡查模块。数据来源和巡查规则已迁入本项目的 `config/banner-check.json`，不再依赖外部文件夹。

### Banner 快路径提交

`执行 Banner 后台配置` 现在默认使用快路径：

1. 用 Playwright 打开真实浏览器完成商城后台登录，并从 `https://shop.ezvizlife.com/templates/index` 进入启用状态的 Homepage Visual Editor。
2. 从页面 DOM 读取 `.home-banner.js-widget-wrapper` 的 `widget-id`、`tpl_id`、`widget_type`，不再依赖悬浮后点击 Edit 按钮。
3. 调用 `https://shop.ezvizlife.com/pages/edit-widget` 读取当前 Banner 列表。
4. 调用 `https://fs.ezvizlife.com/upload.php` 上传 PC / Mobile 图片，上传参数为 `app=mall`、`flag=op_image`、`quality=100`。
5. 将新 Banner 插入目标位置：普通站点第 1 位，波兰站第 2 位；同时按当前位置修正所有站内 Banner 链接的 UTM。
6. 调用 `https://shop.ezvizlife.com/pages/save-widget` 保存完整 Banner 列表。
7. 如果勾选 `上传后发布`，调用 `https://shop.ezvizlife.com/pages/save-all` 发布页面。
8. 保存后继续执行站点巡查；UTM 问题会自动修复，失效链接只返回结果。

### 站点选择

主要按钮：

- `加载站点`：从配置中读取站点列表。
- `全选站点`：勾选全部站点。
- `清空站点`：取消全部站点。

账号字段：

- `商城账号`
- `商城密码`

如果留空，系统会按当前站点从网站账号密码 Excel 表中读取账号。

### Banner 数据

主要字段：

- `标题 Headline`：Banner 主标题。
- `链接 Link`：点击跳转 URL。
- `副标题 Slogan`：Banner 副标题。
- `型号 Model`：产品型号。
- `简介 Introduction`：简介文案。
- `字体颜色`：White 或 Black。
- `轮播位置`：只读提示，普通站点第 1 位；波兰站点第 2 位。
- `上线时间 UTC`：选择上线时间。
- `下线时间 UTC`：选择下线时间。
- `PC 图片`：PC Banner 图片。
- `Mobile 图片`：移动端 Banner 图片；不选时部分流程会使用 PC 图片。

勾选项：

- `隐藏 More 按钮`
- `新窗口打开`
- `上传后发布`

主要按钮：

- `生成 Banner 清单（不提交后台）`
- `执行 Banner 后台配置`

Banner 后台配置执行逻辑：

1. 根据勾选站点读取账号并登录商城后台。
2. 进入 `https://shop.ezvizlife.com/templates/index`。
3. 点击 Homepage。
4. 找到启用状态的 Homepage Visual Editor。
5. 鼠标悬停 `home-banner js-widget-wrapper` 区域，点击 Edit。
6. 新增 Banner slide。
7. 自动移动位置：
   - 普通站点移动到第 1 位。
   - 波兰站点移动到第 2 位，因为波兰第 1 位固定。
8. 填写标题、链接、文案、时间、颜色等字段。
9. 上传 PC / Mobile 图片。
10. 按实际排序修正所有 Banner 的 UTM。
11. 保存。
12. 如果勾选 `上传后发布`，自动点击 Publish。
13. 发布后自动巡查当前站点 Banner 链接。
14. 如果发现 UTM 命名问题，自动进入后台修复并复查。
15. 如果发现链接失效，只把坏链结果返回，不自动改链接。

Banner UTM 规则：

- 内部 EZVIZ 域名会自动添加或覆盖 UTM。
- 外部域名保持原链接，不自动添加 UTM。
- Banner 规则：
  - `utm_source={siteCode}_banner`
  - `utm_medium=banner{position}`
  - `utm_campaign=web_{siteCode}_banner`

示例：

```text
https://www.ezviz.com/id/product/example/123
```

ID 站第 1 位会生成：

```text
https://www.ezviz.com/id/product/example/123?utm_source=id_banner&utm_medium=banner1&utm_campaign=web_id_banner
```

波兰站第 2 位会生成：

```text
https://www.ezviz.com/pl/product/example/123?utm_source=pl_banner&utm_medium=banner2&utm_campaign=web_pl_banner
```

外部链接示例：

```text
https://example.com/landing?a=1
```

会保持原样。

- `执行 Popup 后台配置`

Popup UTM 规则：

### 巡查

用于检查首页活动链接的可达性和 UTM 命名。

主要字段：

- `巡查类型`：
  - `banner`
  - `popup`
  - `banner + popup`
- `Popup 等待 ms`：等待 Popup 出现的时间，默认 5000。

主要按钮：

- `开始巡查`：按所选站点执行 rendered 巡查。
- `修复 Banner UTM`：对单个站点的 Banner UTM 进行后台修复。

进度显示：

- 点击 `开始巡查` 后，系统会创建后台巡查任务。
- 每完成一个站点，结果栏会实时追加进度。
- 进度内容包含当前站点序号、站点名称、站点代码、Banner / Popup 链接数量和首页错误状态。
- 巡查结束后，结果栏会追加易读的问题清单，不直接展示原始 JSON。

巡查结果包含：

- 检查站点数量。
- Banner / Popup 链接数量。
- 坏链数量。
- 内部链接 UTM 命名错误数量。
- 具体问题所属站点、Banner / Popup 类型、Banner 位置、状态码、当前链接、建议修正 URL。

坏链处理规则：

- 坏链只返回结果，不自动替换。
- UTM 命名问题可以自动修复。

## 后台产品替换

当前开放第一阶段的只读能力，用于读取后台商品编辑页 `Details` 标签下的两个字段：

- `Overview`：读取 `vm.pcView.summary` 的完整 HTML。
- `Specifications`：只读取 `vm.pcView.customs` 中名称为 `Specifications` 的字段 HTML。

使用方式：

1. 选择国家站点并填写产品名称。
2. 商城账号密码可以留空，系统会按所选站点从网站账号密码表读取。
3. 点击 `读取 Overview / Specifications`。
4. 系统会强制按所选站点重新登录后台、确认当前身份、定位产品并切换到 `Details`。
5. 页面分别展示 Overview HTML、Specifications HTML、Goods ID、编辑页和读取日志。

只读接口：

```text
POST /api/product-replacement/detail
Content-Type: application/json

{
  "sites": ["hq"],
  "productName": "CP8"
}
```

该接口不会点击 `Complete`，不会修改 Basic Information、Product Specifications、Mobile Details、Detailed Parameters 或 Additional Information，也不会发送保存请求。

## 2026-07-04 更新：Popup 快路径提交

Popup 后台配置已接入“Playwright 登录态 + 接口提交”的快路径，默认不再完整模拟表单点击。

执行流程：
1. 点击 `执行 Popup 后台配置` 后，系统先用 Playwright 打开真实浏览器，复用当前商城后台登录态；如账号不匹配，再走退出和重新登录。
2. 进入 `https://new-shop.ezvizlife.com/popup/edit` 建立 new-shop 登录态。
3. 调用 `https://sgpshop-api.ezvizlife.com/system/get-fs-token` 获取 Popup 图片上传 token。
4. 调用 `https://fs.ezvizlife.com/upload.php` 上传图片，参数使用 `app=mall`、`flag=lottery`、`quality=100`、`type=file`。
5. 调用 `https://sgpshop-api.ezvizlife.com/shop-config/create` 创建 Popup，核心参数为 `moduleType=popup`、`configType` 和 `content`。
6. 如果勾选 `提交后启用`，系统会反查 Popup 编号 `configNo`，再调用 `https://sgpshop-api.ezvizlife.com/shop-config/switch` 启用。

结果栏现在会显示站点、Popup 名称、配置编号、Web/Mobile 链接、图片地址、启用状态和关键日志，不再只展示原始 JSON。

## 2026-07-04 更新：Popup 单资源位规则

Popup 后台目前按单资源位处理。执行 `执行 Popup 后台配置` 时，系统会先检查已有 Popup 资源位：

1. 如果没有旧 Popup，直接创建新 Popup。
2. 如果已有旧 Popup，读取旧 Popup 的下线时间。
3. 如果旧 Popup 下线时间早于当前操作时间，先调用后台删除接口删除旧 Popup，再创建新 Popup。
4. 如果旧 Popup 未过期，停止创建并在结果栏报告旧 Popup 名称、编号和下线时间。
5. 如果旧 Popup 删除请求失败，或删除后仍能在列表中查到旧资源位，停止创建并把失败原因返回到结果栏。

这个检查发生在图片上传之前，因此旧资源位未释放时不会先上传新图片。
## 2026-07-04 更新：语言包上传模块

左侧导航新增 `语言包上传` 模块。该模块用于把 `.xls / .xlsx` 语言包上传到商城后台 Language Management。

使用方式：
1. 点击 `加载站点`，只勾选一个目标站点。
2. 商城账号和密码可留空，系统会按站点从网站账号密码 Excel 自动读取；也可以手动填写覆盖。
3. 填写语言代码，例如 `en-US`。如果留空，系统会尝试从文件名开头自动识别，例如 `en-US.xlsx`。
4. 选择语言包文件。
5. 点击 `上传语言包`。

后台执行逻辑：
- 先复用或登录商城后台。
- 打开 `https://shop.ezvizlife.com/language/index` 建立语言管理页面登录态。
- 直接提交 `POST https://shop.ezvizlife.com/language/upload`。
- multipart 字段包含 `lang_code` 和 `file`。

## 2026-07-04 更新：i18n 重复文案复用

`i18n 语言转换` 现在支持复用已上传语言包里的字段。先上传并解析语言包表格后，再解析 HTML：如果页面文案和语言包中已有 value 完全一致，系统会直接复用原来的 key，例如 `goods.xxx`，不会重新生成新的字段。没有命中的文案才会按当前前缀生成新 key。同一轮解析中重复出现的新文案也会共用同一个新 key。

## 2026-07-04 更新：WTB 产品购买链接配置

`Banner / Popup / 巡查` 页面新增 `WTB 产品购买链接` 区块，用于配置商品详情页 `Additional Information` 中的 `wheretobuy Settings`。

后续调整：WTB 已从 `Banner / Popup / 巡查` 中拆出，成为左侧导航里的独立模块 `WTB 产品购买链接`。该模块拥有独立的站点选择、商城账号密码输入、清单生成按钮、后台执行按钮和结果栏。

使用方式：
1. 先在站点区域只勾选一个目标站点。
2. 可以直接填写单个产品的 `产品名称`、`购买平台`、`购买链接` 进行测试。
3. 也可以先点击 `下载 WTB 信息模板`，填写 `Product / Product Page URL / Channel / Purchasing Link` 后上传批量处理；系统仍兼容中文 `产品名称 / 购买平台 / 购买链接` 和旧英文表头。
4. 点击 `生成 WTB 清单（不提交后台）` 只检查表格识别结果，不会保存后台。
5. 点击 `执行 WTB 后台配置` 会先登录真实商城后台并确认当前用户身份，按产品名读取完整产品模型，然后通过后台接口直接保存购买链接。

当前后台动作：
- 账号密码沿用站点选择逻辑，从网站账号密码 Excel 自动读取，也可以用页面上的商城账号密码手动覆盖。
- 一次真实提交只允许选择一个站点，避免账号和站点不匹配。
- Excel 中同一产品的多行平台链接会自动合并为一次产品保存。
- WTB 使用已登录会话直接调用 `https://shop.ezvizlife.com/goods/do-edit-goods`，不再依赖 `Complete` 按钮和请求监听。
- 每次保存后会重新读取产品编辑数据，逐个平台核对购买链接；回读不一致会作为失败写入结果和报告。
- TDK 同样先确认后台用户身份，再使用该登录会话直接调用 TDK 接口；身份读取失败时不会发送配置请求。


## 2026-07-23 更新：TDK 国家站点选择

TDK 页面会从项目站点配置中加载所有启用站点。上传 Excel 前需要选择一个目标国家站点，校验时会检查每行 `Url Path` 是否属于所选站点；校验通过后，确认框、执行日志和结果都会显示该站点名称与代码。一次提交只处理一个站点。
