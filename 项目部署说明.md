# Office Software 项目部署说明

更新日期：2026-07-01

本文档用于在新电脑上部署 `office-software` 本地操作平台。

## 1. 项目用途

`office-software` 是一个本地 Web 操作平台，当前包含：

- 一键内联打包
- i18n 语言转换
- 下载链接查询
- PDF 上传转地址
- Spec 参数解析
- Datasheet 生成 Spec 表
- 服务中心资料上传平台
- Banner / Popup 后台配置与巡查

项目启动后通过本地浏览器访问：

```text
http://localhost:3217/inline-packager.html
```

## 2. 推荐目录结构

建议在新电脑上保持与当前电脑一致的路径：

```text
E:\office-software
E:\Website-backend\backend-operations
```

其中：

- `E:\office-software`：本地操作平台主项目。
- `E:\Website-backend\backend-operations`：Banner / Popup 巡查、UTM 规则、后台自动化参考文档和脚本。
- 桌面的 `网站账号密码（HQ数字营销组使用）.xlsx`：后台账号密码表，商城后台自动登录会读取它，不要放进 Git 仓库。

如果新电脑没有 E 盘，可以放到其他目录，但需要同步修改 `server.js` 中的固定路径，例如：

```js
const WEBSITE_BACKEND_ROOT = "E:\\Website-backend\\backend-operations";
```

## 3. 必装软件

新电脑需要先安装：

- Node.js LTS
- npm，随 Node.js 一起安装
- Chrome 或 Chromium
- PowerShell，Windows 自带即可

安装后在 PowerShell 中检查：

```powershell
node -v
npm -v
```

能正常输出版本号即可。

## 4. 拷贝项目文件

把以下目录和文件从旧电脑复制到新电脑：

```text
E:\office-software
E:\Website-backend\backend-operations
```

至少需要包含：

```text
E:\office-software\server.js
E:\office-software\package.json
E:\office-software\package-lock.json
E:\office-software\办公软件\111\inline-packager.html
E:\office-software\办公软件\111\src\
%USERPROFILE%\Desktop\网站账号密码（HQ数字营销组使用）.xlsx
E:\Website-backend\backend-operations\website-audit\
E:\Website-backend\backend-operations\ezviz-shop-automation\
```

不必须拷贝的目录：

```text
E:\office-software\node_modules
E:\office-software\runtime_uploads
E:\office-software\.pw-ecadmin-auto-profile
E:\office-software\.pw-ecadmin-profile
E:\office-software\.pw-ezviz-shop-profile
```

这些目录可以在新电脑重新生成。

## 5. 安装依赖

进入项目目录：

```powershell
cd E:\office-software
```

安装 npm 依赖：

```powershell
npm install
```

安装 Playwright 浏览器：

```powershell
npx playwright install chromium
```

安装 Playwright CLI：

```powershell
npm install -g @playwright/cli@latest
```

Banner / Popup 巡查脚本会使用 Playwright CLI，因此需要确保它可用。

## 6. 启动项目

在项目目录执行：

```powershell
cd E:\office-software
npm start
```

看到类似输出说明启动成功：

```text
Office software platform is running at http://localhost:3217/inline-packager.html
```

然后打开：

```text
http://localhost:3217/inline-packager.html
```

## 7. 验证服务状态

浏览器访问：

```text
http://localhost:3217/api/health
```

如果返回：

```json
{"ok":true}
```

说明服务正常。

也可以在 PowerShell 中检查：

```powershell
Invoke-RestMethod http://localhost:3217/api/health
```

## 8. 首次使用检查

### 8.1 账号密码表

确认文件存在：

```text
%USERPROFILE%\Desktop\网站账号密码（HQ数字营销组使用）.xlsx
```

Banner / Popup 后台配置默认会根据站点从这个表中读取账号密码。

如果读取失败，可以在页面中手动填写商城账号和密码。

### 8.2 Website-backend 路径

确认以下目录存在：

```text
E:\Website-backend\backend-operations\website-audit
E:\Website-backend\backend-operations\ezviz-shop-automation
```

如果 Banner / Popup 巡查报错找不到脚本，优先检查这个路径。

### 8.3 Playwright CLI

检查 Playwright CLI：

```powershell
playwright-cli --help
```

如果命令不可用，重新安装：

```powershell
npm install -g @playwright/cli@latest
npx playwright install chromium
```

## 9. 常用功能验证

### 9.1 页面是否能打开

访问：

```text
http://localhost:3217/inline-packager.html
```

左侧能看到功能导航即可。

### 9.2 Banner / Popup 站点能否加载

进入 `Banner / Popup / 巡查`，点击：

```text
加载站点
```

如果站点列表正常显示，说明 `Website-backend` 配置读取成功。

### 9.3 Banner 巡查是否可用

选择一个站点，例如 ID，巡查类型选择 `banner`，点击：

```text
开始巡查
```

如果输出中出现链接数量、坏链数量、UTM 问题数量，说明巡查链路可用。

## 10. 后台自动化说明

部分功能会打开真实浏览器执行后台操作，例如：

- 服务中心资料上传平台
- Banner 后台配置
- Popup 后台配置
- Banner UTM 修复

执行期间注意：

- 不要关闭自动打开的浏览器。
- 如果页面停在登录页，可先确认账号密码表是否正确。
- 如果需要切换账号，系统会尝试在后台右上角用户名位置悬停并点击 Exit，然后重新登录。
- 自动化浏览器 profile 会在新电脑重新生成，不需要从旧电脑拷贝。

## 11. Banner / Popup 依赖说明

Banner / Popup 模块依赖：

```text
E:\Website-backend\backend-operations\website-audit\config\banner-check.json
E:\Website-backend\backend-operations\website-audit\scripts\check-homepage-campaign-rendered.mjs
E:\Website-backend\backend-operations\ezviz-shop-automation\
```

Banner UTM 规则：

```text
utm_source={siteCode}_banner
utm_medium=banner{position}
utm_campaign=web_{siteCode}_banner
```

Popup UTM 规则：

```text
utm_source={siteCode}_popup
utm_medium=popup
utm_campaign=web_{siteCode}_popup
```

内部 EZVIZ 域名会自动添加或覆盖 UTM；外部域名保持原样。

## 12. 常见问题

### 12.1 端口被占用

如果 `3217` 端口被占用，需要先关闭旧进程，或修改 `server.js` 中的端口配置。

查看 Node 进程：

```powershell
Get-Process node
```

结束 Node 进程：

```powershell
Get-Process node | Stop-Process -Force
```

然后重新启动：

```powershell
npm start
```

### 12.2 页面打不开

先检查服务是否启动：

```powershell
Invoke-RestMethod http://localhost:3217/api/health
```

如果无法访问，回到项目目录重新启动：

```powershell
cd E:\office-software
npm start
```

### 12.3 巡查失败

优先检查：

- `E:\Website-backend\backend-operations` 是否存在。
- `playwright-cli --help` 是否可用。
- 网络是否能访问 EZVIZ 官网。
- 站点配置 `banner-check.json` 是否存在。

### 12.4 后台登录失败

优先检查：

- `网站账号密码（HQ数字营销组使用）.xlsx` 是否存在。
- 当前站点是否能在账号表中匹配到账号。
- 是否需要手动在页面输入账号密码覆盖。
- 后台登录地址是否能正常访问。
